# TAZLARM v2.2.3 default routine test fix

The v2.2.2 source intentionally changed `defaultRoutine()` to an empty list so onboarding begins blank. The old unit test still asserted the retired four-stage default and therefore failed after successful Kotlin compilation.

This build updates the test to require a blank default routine and separately confirms that the optional Must Get Up preset still contains Questions → Barcode → Questions → Photo.
